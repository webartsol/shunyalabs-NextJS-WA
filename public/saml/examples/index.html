Complete Sample Document

A comprehensive SAML document demonstrating all major annotation types.

sample.xml Download XSD 
<?xml version="1.0" encoding="UTF-8"?>
<!-- SAML Sample Document - Demonstrates all annotation types -->
<speak xmlns="http://www.w3.org/2001/10/synthesis"
       xmlns:saml="https://shunyalabs.ai/2026/saml"
       version="1.1" xml:lang="en-US">

    <!-- File and speaker metadata -->
    <saml:metadata>
        <saml:file id="sample-001" duration="45000" task="reading"/>
        <saml:speaker id="SPK01" age-group="adult" severity="moderate"/>
    </saml:metadata>

    <!-- Example 1: Incomplete Syllable Repetition (ISR) -->
    <p>I went to the 
        <saml:repetition type="incomplete-syllable" severity="3" count="3">
            <saml:unit>st</saml:unit>
            <saml:unit>st</saml:unit>
            <saml:unit>st</saml:unit>
        </saml:repetition>store yesterday.
    </p>

    <!-- Example 2: Syllable Repetition (SR) -->
    <p>The 
        <saml:repetition type="syllable" severity="2" count="2">
            <saml:unit>ba</saml:unit>
            <saml:unit>ba</saml:unit>
        </saml:repetition>baby was sleeping.
    </p>

    <!-- Example 3: Prolongation -->
    <p>I <saml:prolongation sound="s" severity="3" position="initial">sssss</saml:prolongation>ee what you mean.</p>

    <!-- Example 4: Inaudible Block -->
    <p><saml:block type="inaudible" sound="consonant" severity="4"/>The meeting starts at noon.</p>

    <!-- Example 5: Non-speech and Background -->
    <p>
        <saml:background type="traffic" level="moderate"/>
        <saml:nonspeech type="clear-throat"/> Excuse me.
    </p>

</speak>

Shorthand to XML Conversion

Incomplete Syllable Repetition

Shorthand:

I ( [R:isyl 2] b- b- ) baby is sleeping.

XML:

<saml:repetition type="incomplete-syllable" severity="2">
    <saml:unit>b</saml:unit>
    <saml:unit>b</saml:unit>
</saml:repetition>baby
Syllable Repetition

Shorthand:

( [R:syl 1] I- I- I- ) want to go.

XML:

<saml:repetition type="syllable" severity="1">
    <saml:unit>I</saml:unit>
    <saml:unit>I</saml:unit>
    <saml:unit>I</saml:unit>
</saml:repetition>want
Prolongation

Shorthand:

I s( [P 3] s- )ee what you mean.

XML:

<saml:prolongation sound="s" severity="3" 
    position="initial">sssss</saml:prolongation>ee
Block

Shorthand:

Let's ( [IBC 4] ) go to the store.

XML:

<saml:block type="inaudible" sound="consonant" 
    severity="4"/>go

Common Errors (What NOT to Do)

These examples show invalid SAML that will fail schema validation.

✗

Severity out of range (must be 0-8)

<saml:repetition type="syllable" severity="9">...  <!-- INVALID: max is 8 -->
✗

Repetition with only 1 unit (minimum is 2)

<saml:repetition type="syllable" severity="2">
    <saml:unit>ba</saml:unit>  <!-- INVALID: needs at least 2 units -->
</saml:repetition>
✗

Missing required attributes

<saml:block type="inaudible" severity="3"/>  <!-- INVALID: missing 'sound' attr -->
<saml:prolongation sound="s">sss</saml:prolongation>  <!-- INVALID: missing 'severity' -->
✗

Invalid enumeration values

<saml:repetition type="word" ...  <!-- INVALID: use 'syllable' or 'incomplete-syllable' -->
<saml:block type="silent" ...  <!-- INVALID: use 'inaudible' or 'audible' -->
