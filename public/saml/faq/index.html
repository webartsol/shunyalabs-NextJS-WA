Frequently Asked Questions

Answers to common questions from annotators, developers, and researchers.

General Questions

What is the difference between "dysfluency" and "disfluency"? +

In this specification:

  • Dysfluency refers specifically to stuttering-related speech interruptions (repetitions, prolongations, blocks). These ARE tagged in SAML.
  • Disfluency refers to normal speech interruptions that everyone produces (um, uh, false starts, phrase repetitions). These are NOT tagged.
Do I need to know SSML to use SAML? +

For annotation work, no. You can use the shorthand notation without any XML knowledge. The converter tool handles the XML generation for you.

For software development, basic SSML/XML understanding is helpful, but the specification provides complete examples.

Can I use SAML for languages other than English? +

Yes. The core framework is language-agnostic. You'll need to document language-specific conventions for:

  • Filler words (what counts as normal disfluency)
  • Syllabification rules
  • Grapheme conventions
What audio software should I use? +

Audacity (free) or Praat (free) are recommended. Any software that displays waveforms and allows millisecond-precision measurement will work.

See our Integration Guides for setup instructions.

Classification Questions

How do I classify "I-I-I want"? +

This is Syllable Repetition (SR). "I" is a monosyllabic word, and when repeated, the unit equals a complete syllable.

( [R:syl N] I- I- I- ) want
Is "um" a dysfluency? +

No. "Um", "uh", "er", "like", "you know" are normal filler words (disfluencies). Do not tag them. Write them in the transcript as plain text.

What if someone repeats a two-syllable word like "maybe-maybe"? +

Do not tag. Multi-syllable word repetitions are considered normal disfluency, not stuttering, even if they occur in a person who stutters.

How do I handle "d-d-do-do-dog" with varying unit lengths? +

If all units are incomplete syllables (even if varying in length), classify as ISR (incomplete-syllable).

Only use Mixed when you have both incomplete AND complete syllable units in the same sequence.

What's the difference between IBC and ABC? +
  • IBC (Inaudible Block Consonant) = complete silence before a consonant-initial word
  • ABC (Audible Block Consonant) = you can hear tension, struggle, or partial sounds before a consonant-initial word

Tip: If in doubt, increase volume and check the waveform for any signal.

How do I classify a block before "honest"? +

Classify by phonetic onset, not spelling. "Honest" begins with vowel sound /o/ (the "h" is silent), so it would be IBV or ABV depending on audibility.

Severity Questions

How do I handle boundary values like exactly 1.0 seconds? +

Boundary values go to the higher rating. Exactly 1.0s = severity 2, exactly 2.0s = severity 3, etc.

What if I can't measure precisely? +

Estimate to the nearest 0.5 seconds. When in doubt, use the lower severity rating. Consistency is more important than precision.

Should I include the target word in repetition duration? +

No. Measure from the start of the first repeated unit to the end of the last repeated unit, excluding the final target word.

Annotation Process Questions

Do I use full XML or shorthand? +

Use shorthand for faster annotation during transcription. Convert to full XML only if required by your project's tooling.

The converter tool handles the conversion automatically.

How do I handle overlapping speech? +

Use the <saml:overlap> element. Annotate dysfluencies in the primary speaker's turn and note overlapping content separately.

What if I hear a dysfluency but can't identify the word? +

Use <saml:unintelligible> with:

  • type="guess" if you can estimate the word
  • type="unknown" if you cannot
How long should my audio chunks be? +

Target 10 seconds (acceptable range: 5-15 seconds). Never break chunks in the middle of a dysfluency event.

Edge Cases

What if someone stutters on a filler word like "um"? +

This is rare but can happen. If there's clear involuntary repetition or blocking on "um" itself (e.g., "u-u-u-um"), you may tag it. Use clinical judgment.

How do I annotate covert stuttering? +

Currently, SAML does not have elements for covert stuttering (internal struggle with no audible dysfluency). If the speaker reports internal struggle but produces fluent speech, there is nothing to annotate. Future versions may address this.

What about secondary behaviors (eye blinks, head movements)? +

SAML v1.0 focuses on audible speech patterns. Secondary behaviors are not currently annotated. Video annotation protocols may be developed separately.

How do I handle speech in a very noisy environment? +

Use <saml:background> to document noise. If dysfluencies are obscured by noise and you cannot reliably identify them, note this limitation in metadata or comments. Do not guess at dysfluencies you cannot hear clearly.
